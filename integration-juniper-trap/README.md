# integration-juniper-trap

Juniper SNMP trap processor for [Rapax](https://rapax.io) network monitoring.

## Overview

Processes SNMP traps from network devices via syslog. Reads `/var/log/messages`, filters for `snmptrap`-tagged entries, parses trap data (JSON and legacy formats), and routes events to Redis streams for alert generation or logging.

### Supported Event Categories

- Battery
- Configuration changes
- Device-state (cold/warm start, reboot)
- Environment
- High-availability / redundancy events
- Hardware (fan, PSU, temperature alarms)
- Link-state (interface up/down, CRC errors)
- Power
- Routing (BGP, OSPF, IS-IS neighbor changes)
- Security (authentication failures, ACL denials)

**Total definitions:** 21 trap/event types

## Quick Start

```bash
# Build the image
docker build -t integration-juniper-trap:latest .

# Run the container
docker run -d \
  --name integration-juniper-trap \
  --hostname "$(hostname)" \
  --network rapax-dev-network \
  -v /var/log/messages:/var/log/messages:ro \
  -v "$RAPAXHOME/lib:/opt/rapax/lib:ro" \
  -v "$RAPAXHOME/etc:/opt/rapax/etc:ro" \
  -v "$RAPAXHOME/logs:/opt/rapax/logs" \
  -v /etc/machine-id:/etc/machine-id:ro \
  -e REDIS_HOST=rapax-redis \
  -e "REDIS_PASSWORD=$REDIS_PASSWORD" \
  integration-juniper-trap:latest
```
